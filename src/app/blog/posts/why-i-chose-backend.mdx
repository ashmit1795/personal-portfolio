I used to call myself a web developer.

Not a frontend developer, not a backend developer — just someone who built websites. During the COVID lockdowns, I spent a lot of time on WordPress, tinkering with blogs, running small communities, trying to understand how the internet worked from the outside.

Then I learned HTML. Then CSS. Then JavaScript. Then React.

And somewhere in there, something happened.

Every time I built a frontend feature, I kept wondering: *where does this data actually come from? What's on the other side of this API call? How does the server know who I am?*

I'd finish a React component and immediately open Postman to poke the API it was calling. I'd get distracted reading about JWT and forget what I was originally supposed to be building.

It took me longer than it should have to realize that the backend was where I actually wanted to be.

---

## What pulled me in

The thing about backend engineering is that it's full of *systems thinking*.

A frontend tells a story. A backend *enforces* one.

The authentication middleware decides who gets in. The database schema decides what's possible. The background job decides what happens while nobody's watching. The API contract decides what the frontend can even ask for.

There's something deeply satisfying about that. About designing systems that work correctly even when the user does something unexpected — or when the server crashes at the wrong moment.

Frontend is visible. Backend is felt.

---

## The questions that wouldn't leave me alone

I became obsessed with the questions nobody seemed to be answering in tutorials:

- What happens after you click "Submit"?
- Where is the data actually stored, and why in that structure?
- How do thousands of people use the same app at the same time?
- What happens when a server crashes mid-request?
- How do deployments happen without taking the app down?
- How do engineers debug something they can't see?

Those questions eventually became the reason I'm studying backend systems, cloud infrastructure, and distributed architecture. Every answer leads to three more questions. I've started to think that's actually the point.

---

## Where I am now

I'm still learning. Heavily.

I've built a few backend systems I'm genuinely proud of, worked a production internship where I shipped code real users depended on, and I'm currently going deep on Linux, Docker, and cloud infrastructure.

Every layer you understand makes the one above it make more sense. That's what keeps me going.
